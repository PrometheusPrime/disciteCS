Errors

An error will cause a program to stop execution or not run at all.

Syntax Errors

Every programming language has rules on how a piece of code is supposed to be written. If a line of code does not conform to Python's syntax rules, it will cause a Syntax Error.

if True
    print("Thats True")
The interpreter will not understand the instructions, therefore the program will not run.

How to fix Syntax Errors

  • If you're using a code editor or IDE with syntax highlighting, it will highlight any syntax errors in your code.
  • If you run code with syntax errors, the error message will show where the error(s) is and suggest how to fix it.

Exceptions

An Exceptions is an error that occurs when running a program (Runtime Error). A Statement's syntax might be correct, but it may still cause an Exception.

a = 5
b = 0
c = a/b
print(c)

The line of code where the error occured will be shown, this is called a traceback. The last line of the error message shows the type of error encountered.

If an exception was not handled the program stop runing immediately, the lest of the code will not be executed.

Exception Handling

The try Statement is used to execute code that might cause an error. The except clause defines what to do if an error occurs. If an exception is raised in the try clause, the rest of the statements will not be executed and execution goes to the except block. If no exception occurs, the except clause is skipped. 

try:
    code that might be problematic
except:
    what to do if an error occurs

a = 5
b = 0
try:
    c = a/b
    print(c)
except:
    print("Something went wrong")

Pyton has different types of Exceptions built-in Exception Class .

a = 5
b = 0
try:
  c = a/b
  print(c)
except Exception:
  print("Something went wrong")

We use an alias to get the description of the error raised from the Exception class.

a = 5
b = 0
try:
  c = a/b
  print(c)
except Exception as err:
  print(f'A {err} error occured')

If we know the type of error that might occur, we can handle that specific error.

a = 5
b = 0
try:
  c = a/b
  print(c)
except ZeroDivisionError:
  print("can not divide by 0")

The raise statement is used to manually raise (force) an exception.

The finally clause is always executed, whether there was an exception or not. It is used to clean up (release resources, close files ,etc). 

try:
    code that might be problematic
except:
    what to do if an error occured
else:
    what to do if no error occured in try block
finally:
    what to do, whether there was an error or not
