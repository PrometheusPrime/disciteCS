Errors

Errors in code will cause a program to stop execution or not compile at all.

Syntax Errors

Syntax Errors are a violation of Java's synstax rules.

if True{ 
    System.out.print("That's True)
};
The compiler will not understand the instructions, therefore the program will not compile (Compile Time Errors).

Any code editor or IDE with syntax highlighting will highlight any syntax errors in your code and suggest how to fix them.

Exceptions

An Exceptions is an error that occurs when running a program (Runtime Error). If an exception was not handled the program stops runnng immediately, the lest of the code will not be executed. like dividing by zero. 
int a = 6;
int b = 0;

System.out.print(a/b);

System.out.println("After problematic statements");

Exception Handling

The try Statement is used to execute code that might cause an error. The catch clause defines what to do if an error occurs. If an exception is raised in the try clause, the rest of the statements in try will not be executed and execution goes to the catch block. If no exception occurs, the except clause is skipped and the lest of the code will be executed. 

int a = 6;
int b = 0;

try{
    System.out.print(a/b);

}catch(Exception err){
    System.out.print(err);
}
System.out.println("After problematic statements");

Scanner inputScanner = new Scanner(System.in);

try {

    System.out.print("Enter a Number : ");
    int num1 = inputScanner.nextInt();

    System.out.print("Enter another Number : ");
    int num2 = inputScanner.nextInt();

    double num3 = (double) num1 / num2;
    System.out.println(num3);

} catch (InputMismatchException e) {

    System.out.println(e);

} catch (Exception er) {

    System.out.println(er);
}

The finally block is always executed, whether there was an exception or not. It is used for clean up. For example if created a thread to execute some code, whether that code executed successfully or not you might want to close that thread. 

Scanner inputScanner = new Scanner(System.in);

try {

  System.out.print("Enter a Number : ");
  double num1 = inputScanner.nextDouble();

  System.out.print("Enter another Number : ");
  double num2 = inputScanner.nextDouble();

  double num3 = num1 / num2;
  System.out.println(num3);

} catch (InputMismatchException e) {

  System.out.println(e);

} catch (Exception er) {

  System.out.println(er);

} finally {

  inputScanner.close();

}

Methods and Exceptions

If a method might cause an exception, after parenthesis add the throws keyword followed by the exception or list of exceptions. And if a method that throws an exception is called in another method, that method too must throw an exception.

static double divide(int a, int b) throws ArithmeticException{
    return a / b;
}

Throw Exceptions

The throw keyword is used to explicitly throw an exception.

static double divide(int a, int b) throws ArithmeticException{
    if(b == 0){
        throw new ArithmeticException("Can not divide by 0");
    }
    return a / b;
}
