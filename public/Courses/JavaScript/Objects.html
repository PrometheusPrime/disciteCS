Objects

An Object is a collection of related properties in key(or name) - value pairs.

Don't use numbers as first characters of the name, names that are not strings are automatically converted to string. A value can be any type, other objects or even a function. A property whose value is a function is called a method

Create empty Object by assigning the identifier to an empty set of curly brackets or using the Object constructor.

const employee = {};
//Or
const employee = new Object();
Add or change object properties. 
employee.name = 'John';
employee.age = 45;
employee.position = "CEO";
employee.positionsHeld = ["CFO","President","Board Chairperson"];
employee.greets = function(){
    console.log(`${this.name} says Hi`);
};
console.log(employee);
Access the properties of an object with the dot-notation 
console.log(employee.name);
employee.greets();
If a property name has a special character like a hypen or space, use bracket notation to set or access it. 
employee['Personal Info'] = `Name : ${employee.name}\nAge: ${employee.age}`;

console.log(employee['Personal Info']);

Constructor function


We've looked at diferrent ways of creating an object and working with it's properties. Suppose we want to create another employee object, we'll have to use a different identifier like employee2 and copy and paste the code from our initial employee object. Instead we can define an object type (prototype) employee by creating a constructor function and use it to create employee objects. It is a common convention to capitalise the first letter of the Constructor function.

function Employee(name, age, position, positionsHeld, address) {
    this.name = name;
    this.age = age;
    this.position = position;
    this.positionsHeld = positionsHeld
    this.address = address
    this.personalInfo = function () {
        return `${this.name} ${this.age}`;
    }
}



Create a new object from the prototype with the new keyword.

    
let employee = new Employee('John','Doe',45,[]);
let employee2 = new Employee('John','Doe',45,[])

    
We can still add new properties later to individual objects and not to the prototype.

Frozen Objects


Freezing an object makes it immutable. New properties can not be added and existing properties' values can not be modified unless they are un frozen objects.

Check if an object is frozen.

Object.isFrozen(myObject)


Freeze an object

Object.freeze(myObject)
