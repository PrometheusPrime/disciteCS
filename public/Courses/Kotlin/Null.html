Null Safety

When you declare a variable's type explicitly, it's value can not be null. All types in Kotlin are nun-null by default. 

var userName: String  = null

Add a ? after the type to declare a nullable variable, a variable that can store null as value.

var userName: String? = null

You can not call a method or access a property on a nullable variable.

var userName: String? = null
val startsWithJ = userName.startsWith('J', true)
println(startsWithJ)

Use the !! (not-null assertion operator) to forcebly call a method on a nullable variable, a NullPointerException will be raised if it is null.

var userName: String? = null
val startsWithJ = userName!!.startsWith('J', true)
println(startsWithJ)

Use the ?. (safe call operator) to only call a method on a nullable variable if it isn't null. If it is null, null is returned.

var userName: String? = null
var startsWithJ = userName?.startsWith('J', true)
println(startsWithJ)

userName = "John"
startsWithJ = userName?.startsWith('J', true)
println(startsWithJ)

Use the ?: (Elvis operator) to only use a nullable's value if it is not null, define an alternate value after the operator to use if it's value is null.

var userName: String? = null
var numberOfCharacters = userName?.length ?: 0
println(numberOfCharacters)

userName = "John"
numberOfCharacters = userName?.length ?: 0
println(numberOfCharacters)

If a function you're defining might return null, add a ? after the return type.

fun userInput(): String? {
    var someTxt = readLine()
    return someTxt
}
