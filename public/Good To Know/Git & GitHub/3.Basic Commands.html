Basic Git Commands To Get You Started

This is not a complete list of git commands, but a list of command mostly used..

  • git init creates a repository in the current directory.
  • git status shows the status of the repository.
  • git add adds a file in the working directory to the staging area.
    git add Hello.java .
    To add multiple files in the working directory to the staging area ...
    git add fileA fileB fileC
    git add * adds all the files in the working directory to the staging area.
    git add *.java adds all the files with a specified file extention in the working directory to the staging area.
  • git reset removes all files (changes) from the staging area, but will remain in the working directory with all the changes made.
    git reset file removes a file from the staging area, but it will remain in the working directory with all the changes made to it. You can also unstage multiple files(changes)...
    git reset fileA fileB
  • git commit adds a file(s) from the Staging area to the Repository. Each commit is given a unique 40 character ID. To add a commit message in the command-line use ...
    git commit -m "your commit message" If you have commited the files before, you can directly commit any changes without first adding to the staging area with the -a option ..
    git commit -a
    git commit -am "your commit message" To add a commit message in the command-line .
  • git log shows commit history.
  • git diff compares changes in the Working Directory and the Staging Area. If there are no differences the command will not return anything.
    git diff --staged compares changes in the Staging Area and the Index
    git diff commitID anotherCommitID compares changes between commits.
  • git checkout When you checkout a commit, every thing in the working tree/working directory will be restored to how they were at that point in time..
    git checkout commitID
  • git branch without any argument git branch shows a list of existing branches ,current branch will be marked with an asterisk (*).
    To create a branch run git branch with an argument being the name of the branch you wish to create.... git branch branchName
  • git switch used to switch between branches...
    git switch branchName
    git checkout is also
  • git show shows the most recent commit on the current branch.
  • git merge combines changes from the named branch into the current one..
    git merge branchName.
  • git rebase combines changes between branchescombines changes from the named branch into the current one in a linear sequence..

    • When you make changes in the same file(s), on the same line(s) in different branches and try to merge them, a Merge Conflict will occur. A Merge Conflict will also occur when you try to merge branches, where a file(s) was deleted and updated in another.

    To resolve Merge Conflicts ,open the affected file(s) ,look for conflict markers ,leave the version you wish to keep and delete the conflict markers and the other version ,and then add and commit the changes..

    To avoid Merge Conflicts ,before merging branches run git diff branchA branchB to compare Changes between the tips of the branches. If there are Changes on the tips of the branches that may result in Merge Conflicts if you try merging them,fix them , commit changes ,and then you can merge the branches...

  • git remote add remoteRepositoryUrl Adds a remote repository.
  • git clone remoteRepositoryUrl Clones a remote repository into the current directory..
  • git push Pushes changes from the local to the remote repository. If the remote repository was not added, you will need to specify the repository url ...
    git push remoteRepositoryUrl
  • git fetch Fetches changes from a remote repository.
  • git pull Fetches changes from a remote repository and integrates them into the current branch.
